# 디자인 프롬프트 — agent 화면 (Claude Design 붙여넣기용)

> 이 문서는 **Claude Design에 그대로 붙여넣는 프롬프트**다. 아래 본문을 복사해서 사용하면 된다.
> agent 관련 화면/패널의 외형에만 집중한다. **공유 대시보드 셸·메인 통합 뷰는 `docs/design/screen-design.md`(화면 1)를 그대로 따르고 여기서 재정의하지 않는다**(skill 디자인 프롬프트와 중복 방지). 데이터 모델은 `docs/architecture/structure.md`, agent 스키마/손실 규칙은 `docs/verification/agent-format-verification.md`를 근거로 한다.

---

## 목적 (한 줄)

AI-Agent Dashboard에서 **agent 항목**(중립 스키마)을 도구별(Claude/Codex/Gemini)로 보고, 변환 시 무엇이 손실되는지 정직하게 드러내며, 2단계 확인으로 안전하게 동기화하는 화면의 외형을 만든다. **로컬 전용 도구 — 데이터는 외부로 전송되지 않는다.**

---

## 설계 전제 (반드시 지킬 톤·원칙)

- **로컬 전용:** 모든 작업은 `127.0.0.1`/로컬 파일에서만 일어난다. 외부 전송·클라우드 동기화 없음. 헤더/푸터 어딘가에 "로컬 전용 · 외부 전송 없음" 표식을 유지한다.
- **정직한 손실 표시:** 변환 불가 항목은 임의로 채우지 않고 `⚠ 미지원/손실`로 명시한다. 손실은 숨기지 말고 눈에 띄게.
- **안전 우선(2단계):** 모든 쓰기는 `미리보기(diff) → 적용`. 적용은 한 번 더 명시적으로 눌러야 하고, 적용 전 자동 백업 경로를 보여준다.
- **가독성 최우선:** diff와 손실 목록은 색만으로 구분하지 말고 아이콘+라벨을 함께. 정보 밀도는 높되 스캔이 쉬워야 한다.

---

## 도구 3색 토큰 방향 (Claude/Codex/Gemini)

세 도구를 **일관된 컬러 토큰**으로 구분한다(메인 통합 뷰의 열 색과 동일하게 유지 — 셸에서 정의한 토큰을 재사용).

- **Claude** — 따뜻한 코랄/테라코타 계열(브랜드 친화). 토큰명 예: `--tool-claude`.
- **Codex** — 중립 그레이/슬레이트 계열(아웃라이어·TOML). 토큰명 예: `--tool-codex`.
- **Gemini** — 블루/인디고 계열. 토큰명 예: `--tool-gemini`.
- 각 색은 **칩/도트/좌측 액센트 바**로만 쓰고, 본문 텍스트 색으로 남용하지 않는다(가독성).
- **상태 색은 도구 색과 분리한다:** 동기화됨=초록, drift=앰버/주황, 손실=앰버+⚠, 없음=중립 회색, 오류=빨강. 상태 색과 도구 색이 겹쳐 혼동되지 않게 한다.

## 만들 화면/패널 목록 (agent 한정)

1. **A. agent 상세 패널** — 행 클릭 시 우측 슬라이드. 중립 스키마 표시 + 도구별 상태(최신/drift/손실) + 손실(⚠) 구체 표시.
2. **B. agent Sync(push) 흐름** — 대상 도구 선택 → 1단계 미리보기(diff + 손실 목록) → 2단계 적용 확인 → 적용 후 백업 경로 표시.
3. **C. 개인정보(PI) 게이트 모달** — 민감 정보 의심 시 멈추고 확인.

> 메인 통합 뷰(표/필터/검색/서버 상태/빈·로딩·오류 배너)는 `screen-design.md` 화면 1을 따른다. 여기서는 **agent 행을 클릭한 다음부터**의 외형만 정의한다.

---

## A. agent 상세 패널 (우측 슬라이드 패널)

행에서 🤖 agent를 클릭하면 오른쪽에서 미끄러져 들어오는 패널. 좌=canonical 중립 스키마, 우=도구별 상태.

### 텍스트 와이어프레임

```
┌ 🤖 reviewer ───────────────────────────────────────────────[×]┐
│ id: agent-reviewer   타입: agent   scope: global               │
│ 출처(source): codex   수정: 2026-06-30 15:40                   │
│  · 로컬 전용 · 외부 전송 없음                                   │
├───────────────────────────────────┬────────────────────────────┤
│ canonical (중립 스키마)            │ 도구별 상태                 │
│                                    │                             │
│  name         reviewer             │ ● Claude   ✓ 최신           │
│  description  코드 리뷰 보조…       │ ● Codex    ⚠ 손실           │
│  model        claude-opus-4-…      │   └ tools 미지원(전부 손실) │
│  tools        [Read] [Grep] [Bash] │ ● Gemini   ⚠ 손실           │
│               [Glob]               │   └ Glob 매핑 미확인         │
│  system_prompt                     │                             │
│  ┌───────────────────────────────┐ │ [모든 도구로 미리보기]      │
│  │ You are a code reviewer…      │ │                             │
│  │ (스크롤 영역, 모노스페이스)    │ │                             │
│  └───────────────────────────────┘ │                             │
├───────────────────────────────────┴────────────────────────────┤
│ [Claude로 미리보기] [Codex로 미리보기] [Gemini로 미리보기] [Import]│
└──────────────────────────────────────────────────────────────────┘
```

### 컴포넌트

- **패널 헤더:** agent 이름(🤖 + name), 닫기(×). 그 아래 메타 라인: `id` / `타입` / `scope(global·project)` / `출처(source_tool)` / `수정 시각`. 메타는 작은 모노스페이스 라벨-값 쌍.
- **좌측: canonical 스키마 카드.** 필드 순서 고정: `name → description → model → tools → system_prompt`. (옵션 필드 `temperature`/`max_turns`/`mcp_servers`/`source_notes`가 있으면 "추가 필드" 접이식 섹션으로.)
  - `tools`는 **칩(토큰) 목록**으로 렌더. 각 칩은 도구 이름(예: `Read`, `Grep`, `Bash`, `Glob`). 매핑 손실이 있는 칩(예: `Glob`)에는 `⚠` 마이크로 배지를 붙인다.
  - `system_prompt`는 **모노스페이스 + 스크롤 가능한 코드 블록**(가독성 우선, 줄바꿈 유지).
- **우측: 도구별 상태 리스트.** Claude/Codex/Gemini 3행. 각 행 = `[도구 색 도트] 도구명 [상태 배지]`. 상태 배지: `✓ 최신`(초록) / `⚠ drift`(앰버) / `⚠ 손실`(앰버+⚠) / `✗ 없음`(회색).
  - **손실 행은 바로 아래 들여쓴 사유 라인**으로 "무엇이 왜 손실인지" 구체 표시:
    - `Codex: tools 미지원 → tools 전부 손실` (Codex는 에이전트별 tools 필드 없음)
    - `Gemini: Glob 매핑 미확인 → 해당 도구 손실`
    - drift면 `Codex: developer_instructions 본문 다름` 같은 한 줄 요약.
  - 사유 라인은 작은 글씨 + 좌측 앰버 액센트 바. 칩이 여러 개 손실이면 손실된 칩 이름을 나열.
- **푸터 액션 바:** `[Claude로 미리보기] [Codex로 미리보기] [Gemini로 미리보기]`(→ 화면 B), `[Import]`. 각 도구 미리보기 버튼은 해당 도구 색 액센트. "없음" 상태 도구는 미리보기 버튼을 "이 도구로 생성"으로 라벨 변경.

### 상태

- **로딩:** 우측 도구별 상태는 클릭 시 계산되는 하이브리드 모델(D9)이므로, 패널을 열면 좌측 canonical은 즉시 표시되고 우측 상태는 **스켈레톤 3줄 → 채움**. "도구별 상태 확인 중…" 마이크로 텍스트.
- **빈:** tools가 비어 있으면 `tools — (없음)` 회색 처리. system_prompt 없으면 `(본문 없음)` 플레이스홀더.
- **에러:** 특정 도구 상태 계산 실패 시 그 행만 `🔴 확인 실패` + "다시 시도" 마이크로 링크. 패널 전체는 죽지 않음.
- **손실:** 위 사유 라인. 손실이 하나라도 있으면 패널 헤더에 `⚠ 변환 손실 N건` 요약 배지.

## B. agent Sync(push) 흐름 — 미리보기 → 적용

도구 미리보기 버튼을 누르면 시작. **1단계 미리보기**(diff + 손실)에서 한 번, **2단계 적용 확인**에서 또 한 번 — 두 번 눌러야 실제 쓰기.

### 텍스트 와이어프레임

```
1단계: 미리보기 (diff + 손실)            2단계: 적용 확인 (재확인)
┌ Codex로 동기화 미리보기 ────────[×]┐   ┌ 적용하시겠어요? ───────────[×]┐
│ 대상 도구: ● Codex (.toml)         │   │ 항목: reviewer (agent)        │
│ 대상 경로: ~/.codex/agents/        │   │ 대상: ~/.codex/agents/        │
│            reviewer.toml           │   │        reviewer.toml          │
│                                    │   │ 백업: canonical/backups/      │
│ ⚠ 변환 손실 1건                    │   │        2026-06-30T1540/       │
│  · tools 미지원 → [Read][Grep]…    │   │        reviewer.toml.bak      │
│    전부 손실 (Codex엔 tools 필드   │   │                               │
│    없음)                           │   │ ⚠ 변환 손실 1건 — tools 미지원 │
│ ────────────────────────────────── │   │                               │
│ diff (canonical → Codex .toml)     │   │ 이 작업은 위 경로의 파일을     │
│   model = "claude-opus-4-…"        │   │ 덮어씁니다. 적용 전 자동 백업. │
│ + developer_instructions = """     │   │ · 로컬 전용 · 외부 전송 없음   │
│ +   You are a code reviewer…       │   │                               │
│ + """                              │   │      [취소]   [적용 실행]      │
│ - (이전 파일 없음)                 │   └───────────────────────────────┘
│                                    │
│         [취소]      [적용하기 →]   │   적용 후 →  ✅ 완료 토스트
└────────────────────────────────────┘   + 백업 경로 + [status 새로고침]
```

### 컴포넌트

- **미리보기 모달/패널(1단계):**
  - 상단: **대상 도구 칩**(도구 색 + 포맷 표기: Claude `.md`·Gemini `.md`·Codex `.toml`) + **대상 경로**(모노스페이스, 전체 경로).
  - **손실 요약 블록(상단 고정):** `⚠ 변환 손실 N건`을 diff 위에 크게. 항목별 한 줄: 무엇이(필드/도구 이름) 왜(사유) 손실인지. 손실 0건이면 `✓ 손실 없음 — 모든 필드 변환됨` 초록 라인.
    - 예: `tools 미지원 → [Read][Grep][Bash][Glob] 전부 손실 (Codex엔 에이전트별 tools 필드 없음)`
    - 예: `Glob 매핑 미확인 → Gemini 변환 시 해당 도구 제외`
  - **diff 패널:** 추가 라인(+) 초록 배경/좌측 바, 삭제 라인(-) 빨강 배경/좌측 바, 변경 없는 컨텍스트는 중립. **모노스페이스, 줄 단위, 가로 스크롤 가능.** 추가/삭제는 색 + `+`/`-` 기호 둘 다(색맹 대비). 새로 생성이면 "이전 파일 없음" 컨텍스트 라벨.
  - 푸터: `[취소]` / `[적용하기 →]`. 적용하기는 **이 단계에서 쓰기를 하지 않고** 2단계 모달을 띄운다.
- **적용 확인 모달(2단계):**
  - 항목명 + 대상 경로 + **백업 경로**(`canonical/backups/<timestamp>/…`)를 명확히 표기.
  - **손실 건수 재고지**(`⚠ 변환 손실 N건`).
  - "이 파일을 덮어씁니다 · 적용 전 자동 백업" 경고 문구 + 로컬 전용 표식.
  - 푸터: `[취소]` / `[적용 실행]`. 적용 실행 버튼은 **위험 강조 색(채워진 버튼)**, 취소는 외곽선.
- **적용 후 결과:**
  - 성공: `✅ 완료` 토스트 + 백업 경로 표시 + `[status 새로고침]` 액션.
  - 실패: `🔴 적용 실패` + 원인 한 줄 + "백업으로 복구" 안내(백업 경로 재표기).

### 상태

- **로딩:** 미리보기 진입 시 diff 계산 중 스켈레톤 + "변환·diff 계산 중…". 적용 실행 시 버튼 스피너 + 비활성화(중복 클릭 방지).
- **빈:** diff가 비어 있으면(=이미 동일) `✓ 변경 없음 — 이미 최신` 안내 + 적용 버튼 비활성.
- **에러:** 변환 실패 시 모달 안에 빨강 에러 박스(diff 자리). 적용 실패는 위 결과 참고.
- **손실:** 손실 요약 블록(상단). 손실이 있어도 적용은 가능하되, 2단계에서 건수를 다시 보여주고 동의 후 진행.

## C. 개인정보(PI) 게이트 모달

Import(pull)이나 스캔 본문에서 민감 정보(이메일/토큰/시크릿 키워드)가 의심되면 **멈추고** 확인받는 모달(D7). agent의 `system_prompt`/`source_notes`에 민감 내용이 섞일 수 있어 agent 흐름에서도 동일 적용.

### 텍스트 와이어프레임

```
┌ ⚠ 민감 정보가 감지되었습니다 ─────────────────────────[×]┐
│ 항목: reviewer (agent) · 필드: system_prompt              │
│ 감지: 이메일 형태 1건, 토큰 키워드 1건                    │
│ ─────────────────────────────────────────────────────────│
│ 해당 줄 (마스킹):                                          │
│   …contact ****@****.com for…                             │
│   …API_KEY=sk-************…                               │
│ ─────────────────────────────────────────────────────────│
│ 어떻게 처리할까요?                                         │
│   ○ 가명/더미로 대체    ◉ 이 항목 제외(권장)   ○ 그대로 진행│
│ · 로컬 전용 · 외부 전송 없음                               │
│                              [취소]        [확인]          │
└───────────────────────────────────────────────────────────┘
```

### 컴포넌트

- 헤더: `⚠ 민감 정보가 감지되었습니다`(앰버/경고 톤, 위험은 아님 — 주의 단계).
- 컨텍스트: 항목명 + 어떤 필드에서(예: `system_prompt`) 감지됐는지.
- 감지 요약: 종류·건수(이메일/토큰/시크릿 키워드 등).
- **해당 줄(마스킹):** 모노스페이스, 민감부는 `****` 마스킹. **원문 노출 금지.**
- 선택지(라디오): `가명/더미로 대체` / `이 항목 제외` / `그대로 진행`. **기본값은 보수적으로 "이 항목 제외(권장)"**.
- 푸터: `[취소]` / `[확인]`.

### 상태

- **로딩:** 거의 즉시(로컬 패턴 탐지). 필요 시 미세 지연만.
- **빈:** 감지 0건이면 이 모달은 뜨지 않고 흐름이 통과. (이 화면 자체가 "감지됨" 전제)
- **에러:** 마스킹 실패 등은 안전 우선 — `그대로 진행`을 비활성화하고 "제외"만 허용.
- **손실/주의:** "그대로 진행" 선택 시 작은 경고 텍스트로 재고지(이 내용이 canonical에 저장됨).

## 반응형

- **넓은 화면(≥1024px):** 상세 패널 A는 좌(canonical)/우(도구별 상태) **2열**. 메인 표 위에 우측 슬라이드 패널(약 480–560px 폭) 오버레이.
- **중간(640–1023px):** 상세 패널은 표를 거의 덮는 시트. canonical/도구별 상태를 **세로 스택**(canonical 먼저, 그 아래 도구별 상태). diff는 가로 스크롤 유지.
- **좁은 화면(<640px):** 패널/모달은 **풀스크린 시트**. tools 칩은 줄바꿈 래핑. diff·system_prompt 코드 블록은 가로 스크롤. 푸터 액션 버튼은 세로 스택(주 액션이 위/강조).
- 모든 모달(B 2단계, C)은 모바일에서 **하단 시트** 형태 허용.

## 컴포넌트 요약 (재사용 토큰)

- **도구 칩/도트:** Claude/Codex/Gemini 3색 + 포맷 라벨(`.md`/`.md`/`.toml`).
- **상태 배지:** `✓ 최신`(초록) / `⚠ drift`(앰버) / `⚠ 손실`(앰버+⚠) / `✗ 없음`(회색) / `🔴 오류`(빨강).
- **손실 사유 라인:** 앰버 좌측 바 + 작은 글씨 + `⚠`. "무엇 → 왜" 형식.
- **tool 칩:** PascalCase(Claude)/snake_case(Gemini) 이름, 매핑 미확인 칩엔 `⚠` 마이크로 배지.
- **diff 라인:** `+`(초록)/`-`(빨강)/컨텍스트(중립), 기호+색 병행, 모노스페이스, 가로 스크롤.
- **경로 표기:** 모노스페이스, 대상 경로/백업 경로는 항상 전체 경로.
- **위험 버튼:** 적용 실행 = 채워진 강조색. 취소 = 외곽선. 2단계 전까지 실제 쓰기 없음.
- **로컬 전용 표식:** 패널/모달 하단에 "· 로컬 전용 · 외부 전송 없음" 마이크로 라인.
