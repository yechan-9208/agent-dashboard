# Claude Design 프롬프트 — Phase 2 (사용량 · 자기 점검)

Claude Design에 그대로 붙여넣어 Phase 2 화면 외형을 만들기 위한 프롬프트.
공유 대시보드 셸·통합 뷰는 [`screen-design.md`](screen-design.md)를 **참조**만 하고 재정의하지 않는다.
실제 구현된 UI(현재 동작 중)에 맞춰 작성됨.

---

## 목적 (한 줄)
대시보드가 작업을 수행하지 않고 **바깥에서 관찰**한 사용량으로 항목의 상태(노후화)를 보여주고,
유지보수 **제안을 승인/거절**하게 한다. **자동 적용은 절대 없음** — 전부 제안 → 사람 승인.

## 설계 전제
- 관찰 기반: 신호는 **A(편집/변경: git churn·mtime, 견고)** + **B(실제 호출: best-effort, 깨질 수 있음)**. 수치마다 **A인지 B인지 출처를 구분 표시**(B는 신뢰도 낮음 배지).
- 안전: 삭제 없음 → **archive(복구 가능)**. pin한 항목은 자동 노후화 제외. 모든 변경은 staging→승인.
- 자동 스케줄 없음: 점검(review)·동기화(push)는 **사용자가 누를 때만**. 마지막 시각만 표시.

## 화면/패널 (Phase 2 한정)

### A. 상단 상태 줄 — "마지막 동기화/점검 시각"
- 텍스트 한 줄: `마지막 동기화(push) — claude: <시각>  codex: <시각>  gemini: <시각>  |  마지막 점검(review): <시각>`
- 와이어프레임: 모드 토글 줄 아래, 옅은 색 보조 텍스트. 시각은 상대표기(예: "2시간 전") + 절대표기 툴팁 권장.

### B. 사용량 / 상태 테이블 (LRU — 오래 안 쓴 순)
```
┌ 사용량 · 상태  (오래 안 쓴 순) ───────────────────────────────┐
│ id              종류    상태       churn  최근활동      pin   │
│ skill-code-rev  skill   🟢active   3      2일 전        [📌]  │
│ agent-reviewer  agent   🟡stale    0      40일 전       [pin] │
│ instr-global    instr   🔴archived 1      100일 전      [pin] │
└───────────────────────────────────────────────────────────────┘
```
- **상태 배지**: `active`=초록 / `stale`=노랑 / `archived`=빨강(또는 회색). 색은 도구 3색과 **다른 축**(상태색).
- **churn**: 편집 횟수(A). 옆에 작은 출처 배지(A=git/mtime).
- **최근활동**: max(편집, 호출). 호출(B) 데이터가 있으면 작게 "호출 N회(B)" 표기.
- **pin 토글**: 누르면 📌(자동 노후화 제외)↔해제.
- 상태: 로딩(스켈레톤 행), 빈("가져온 항목 없음"), 에러.

### C. 점검(review) 실행
- 버튼 2개: **[점검 실행]**(제안 staging) / **[미리보기만]**(dry-run, staging 없음).
- 실행 후 요약 텍스트: `제안 N개, 상태 {active:x, stale:y, archived:z}`.

### D. 제안 인박스 (카드 + 승인/거절)
```
┌ [archive] 'agent-reviewer' 장기 미사용 → archive ────────────┐
│ 상태 archived. archive 영역으로 이동(복구 가능, 삭제 아님).   │
│                                   [승인]   [거절]            │
└───────────────────────────────────────────────────────────────┘
┌ [consolidate] 유사: code-review ↔ code-reviewer (0.58) ──────┐
│ 본문 유사(Jaccard 0.57) — 통합할지 / 둘 다 둘지 선택.        │
│                                   [승인]   [거절]            │
└───────────────────────────────────────────────────────────────┘
```
- **kind 배지**: `archive`(이동·되돌릴 수 있음) / `consolidate`(중복, 정보형) / `version`(churn↑, 정보형). 정보형은 "기록만, 파괴적 동작 없음" 힌트.
- 승인=초록 계열, 거절=중립. `archive` 승인은 실제 이동이므로 색으로 구분.
- 빈 상태: "대기 중인 제안 없음 — [점검 실행]으로 생성".

### E. (예약) 리포트 뷰
- review 실행 결과(run.json/REPORT.md) 타임라인. 이번 외형 범위 밖이면 placeholder만.

## 공통
- **색 토큰 2축**: ① 도구색(Claude/Codex/Gemini, Phase1과 동일) ② 상태색(active/stale/archived). 섞이지 않게.
- **신뢰도 표시**: B(호출) 수치엔 "추정·깨질 수 있음" 톤(옅게/물음표). A(편집)는 확정 톤.
- **안전 강조**: "자동 적용 없음", archive=복구 가능, pin=보호 를 시각적으로.
- 반응형(넓은/좁은 화면, 테이블 가로 스크롤), 로컬 전용(외부 전송 없음) 헤더 표기 유지.
