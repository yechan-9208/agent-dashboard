# 디자인 프롬프트 — Skill 화면 (Claude Design 붙여넣기용)

> 이 문서는 **Claude Design에 그대로 붙여넣는 프롬프트**다. AI-Agent Dashboard의 **skill 관련 화면/패널**에만 외형을 입히기 위한 입력이다.
> 공유 대시보드 셸(헤더·서버 상태·통합 표·필터/검색)과 통합 뷰는 `docs/design/screen-design.md`에 정의돼 있다 — **여기서는 그 셸을 "참조"만 하고 재정의하지 않는다.** (agent 디자인 프롬프트와 중복 방지)

---

## 목적 (한 줄)

세 CLI(Claude·Codex·Gemini)에 흩어진 skill을 **표준 SKILL.md 코어 + 도구별 메타 오버레이로 분리해서 보여주고**, 충돌·출처·개인정보를 안전하게 다루며 canonical로 가져오는(Import/pull) skill 전용 화면의 외형을 만든다. **로컬 전용 도구**(외부 전송 없음).

---

## 배경 컨텍스트 (디자인이 알아야 할 사실)

- **skill = 표준 `SKILL.md`(YAML frontmatter: `name`·`description` + 마크다운 본문) "그대로" 보존 + 도구별 확장 메타는 "분리" 보관**(코어를 깨지 않음). → 상세 패널은 이 **두 층(코어 / 오버레이)을 시각적으로 분리**해야 한다.
- **Claude skill은 `~/.claude/skills`가 아니라 대부분 플러그인(`~/.claude/plugins/<plugin>/skills/`)에서 온다.** → 출처에 **플러그인 배지**가 필요.
- skill 스캔 중 본문에서 **개인정보/시크릿 의심**이 보이면 멈추고 사용자에게 확인받는다(PI 게이트).
- 같은 이름 skill이 **여러 도구에 동시 존재**할 수 있다 → Import 시 **충돌 선택 UI** 필요.
- 모든 가져오기/쓰기는 안전 흐름을 따른다: 자동으로 전체 본문을 읽지 않고, **사용자가 연 항목만** 내용을 계산한다.

---

## 화면/패널 목록 (이 프롬프트의 범위)

1. **Skill 상세 패널** — 통합 표에서 skill 행 클릭 시 우측 슬라이드. 코어 ↔ 오버레이 분리 + 출처.
2. **Import(pull) 흐름** — 가져올 skill 선택 + 같은 이름이 여러 도구에 있을 때 충돌 선택.
3. **플러그인 출처 배지** — skill 카드/행/패널에 공통으로 쓰는 작은 컴포넌트.
4. **개인정보(PI) 게이트 모달** — 민감 의심 시 멈춤·확인.

> 통합 표(화면 1)·공유 헤더·서버 표시·Sync(push) 흐름은 **`screen-design.md` 참조**. 여기서 새로 그리지 않는다. 단, 통합 표의 **skill 행(`🧩`) 셀**에 위 3번 출처 배지를 얹는 정도만 일관성 차원에서 다룬다.

---

## 화면 1. Skill 상세 패널 (우측 슬라이드 패널)

**목적:** 한 skill의 **표준 코어**(name/description + 본문)와 **도구별 메타 오버레이**를 분리해서, 어디서(어느 도구/어느 플러그인) 왔는지와 함께 보여준다.

### 텍스트 와이어프레임

```
┌ 🧩 code-review ─────────────────────────────────────────[×]┐
│ id: skill-code-review   타입: skill   scope: global         │
│ 출처: ⬣ Claude · 플러그인 「anthropic-skills」   수정: 06-30 │
├─────────────────────────────────────────────────────────────┤
│ ── 코어 (표준 SKILL.md · 모든 도구 공통) ────────────────── │
│  name:        code-review                                   │
│  description: Review the current diff for correctness ...   │
│  ┌ 본문 (마크다운) ─────────────────────────────────────┐  │
│  │ # Code Review                                         │  │
│  │ When invoked, gather the diff and ...                 │  │
│  │ ...                                          (스크롤)  │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                             │
│ ── 도구별 메타 오버레이 (코어와 분리·도구마다 다를 수 있음) │
│  ┌ ⬣ Claude ──────────┐ ┌ ▢ Codex ───────┐ ┌ ◆ Gemini ──┐ │
│  │ ✓ 최신             │ │ ✓ 최신          │ │ ✗ 없음      │ │
│  │ 플러그인:           │ │ allowed-tools:  │ │ (canonical에│ │
│  │  anthropic-skills   │ │  Read, Grep     │ │  서 생성 가능)│ │
│  │ 위치: ~/.claude/    │ │ 위치: ~/.codex/ │ │             │ │
│  │  plugins/.../skills │ │  skills/        │ │ [미리보기]   │ │
│  └─────────────────────┘ └─────────────────┘ └─────────────┘ │
├─────────────────────────────────────────────────────────────┤
│        [Gemini로 미리보기]  [Codex로 미리보기]  [Import]     │
└─────────────────────────────────────────────────────────────┘
```

### 핵심 디자인 요구

- **두 층을 명확히 분리**: 상단 "코어" 섹션과 하단 "도구별 메타 오버레이" 섹션 사이에 강한 구분선/섹션 헤더. 코어는 "모든 도구가 공유하는 원본", 오버레이는 "도구마다 다를 수 있는 부가 메타"라는 위계가 한눈에 보여야 함.
- **코어 섹션**: `name`·`description`은 키-값(label/value)로, 본문(`SKILL.md` 마크다운)은 자체 스크롤되는 읽기 카드. 본문은 **가독성 최우선**(넉넉한 줄간격·코드블록 모노스페이스·적정 폭).
- **오버레이 섹션**: 도구 3개(Claude/Codex/Gemini)를 **나란한 3개 카드**로. 각 카드는 도구 색 토큰 + 상태(✓최신 / ⚠drift / ✗없음 / ⚠손실) + 그 도구 고유 메타(allowed-tools, 위치 경로, 플러그인 등) + 필요 시 작은 액션(미리보기). 없는 도구 카드는 비활성/점선으로 "없음"임을 표시하되 canonical로부터 생성 가능함을 안내.
- **출처(provenance)**: 패널 헤더에 출처 도구 + **플러그인 출처 배지**(아래 화면 3) + 수정일.
- **액션 바**(하단 고정): 도구별 미리보기(→ Sync push 흐름, `screen-design.md` 참조)와 **Import**(→ 화면 2).

### 상태

- **로딩**: 코어/오버레이 영역 스켈레톤(본문 카드 + 3개 오버레이 카드 자리).
- **빈/부분**: 특정 도구에 없으면 그 오버레이 카드만 "없음"으로. canonical에만 있고 어느 도구에도 배포 안 됐으면 오버레이 3개 모두 "없음".
- **에러**: 본문을 읽지 못하면(권한·삭제됨) 코어 본문 카드에 에러 메시지 + 재시도. 오버레이 단위 에러는 해당 카드에만 국한.
- **드리프트(drift)**: 코어가 도구 사본과 다르면 해당 오버레이 카드에 `⚠ drift` 배지 + "무엇이 다른지" 한 줄 요약.

## 화면 2. Import(pull) 흐름 + 충돌 선택

**목적:** 각 도구에 흩어진 기존 skill을 canonical로 가져온다(역방향). 같은 이름이 여러 도구에 있으면 **사용자가 어느 버전을 쓸지 선택**한다.

### 텍스트 와이어프레임

```
┌ Import: 가져올 skill 선택 ──────────────────────────────────┐
│ (skill만 필터됨)                              검색: [______] │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ [✓] 🧩 code-review    ⬣ Claude·플러그인 anthropic-skills │ │
│ │ [✓] 🧩 verify         ▢ Codex / skills/                  │ │
│ │ [ ] 🧩 deep-research   ⬣ Claude·플러그인 (개인) ⚠PI 의심  │ │
│ └─────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│ ⚠ 충돌: 'code-review' 가 여러 도구에 있습니다               │
│ ┌ code-review ──────────────────────────────────────────┐  │
│ │ 어느 버전을 canonical로?                                │  │
│ │  ◉ ⬣ Claude 버전 사용  (플러그인 anthropic-skills)      │  │
│ │  ○ ▢ Codex 버전 사용   (allowed-tools: Read, Grep)      │  │
│ │  ○ 둘 다 보존 (code-review, code-review-codex 로 분리)  │  │
│ │  [▾ 두 버전 diff 비교]                                  │  │
│ └─────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│   3개 선택 · 충돌 1건 미해결 없음   [취소]  [canonical로 가져오기]│
└─────────────────────────────────────────────────────────────┘
```

### 핵심 디자인 요구

- **2영역 구성**: 위 = 가져올 skill 체크리스트(도구/플러그인 출처 배지 + PI 의심 표식), 아래 = **충돌 카드** 영역.
- **충돌 카드**: 같은 이름이 여러 도구에 있으면 카드로 등장. 라디오 선택(도구 A 버전 / 도구 B 버전 / **둘 다 보존**) + 각 옵션에 출처·핵심 차이 한 줄. "두 버전 diff 비교" 펼침(diff는 가독성 최우선 색 구분).
- **"둘 다 보존"** 선택 시 새 canonical 이름이 어떻게 분리되는지 미리 보여줌(예: `code-review` / `code-review-codex`).
- **PI 게이트 연동**: PI 의심(⚠) 항목이 선택돼 있으면 "가져오기"가 곧장 실행되지 않고 **PI 게이트 모달(화면 4)** 을 먼저 띄움.
- **푸터 상태 텍스트**: 선택 개수 + 미해결 충돌 건수. 미해결 충돌이 있으면 "가져오기" 버튼 비활성.

### 상태

- **빈**: 가져올 skill 없음("모든 skill이 이미 canonical에 있음" 안내).
- **로딩**: 도구 스캔 중 체크리스트 스켈레톤(존재/이름만 먼저, 내용은 항목 펼칠 때).
- **충돌 있음**: 충돌 카드 강조(주의색 테두리) + 미해결 시 진행 차단.
- **에러**: 특정 도구 스캔 실패 시 해당 출처 그룹에만 에러 배너 + 재시도(나머지는 진행 가능).
- **성공**: 가져오기 후 ✅ 토스트 + "통합 뷰 새로고침" 안내 + canonical git 커밋 표시.

## 화면 3. 플러그인 출처 배지 (공통 컴포넌트)

**목적:** Claude skill은 대부분 플러그인에서 온다는 사실을 어디서든 한눈에. 통합 표의 skill 행, 상세 패널 헤더, Import 체크리스트에서 **재사용**한다.

### 표현 규칙 (텍스트)

```
출처 형태:
  ⬣ Claude · 플러그인 「anthropic-skills」      ← 플러그인에서 온 skill
  ⬣ Claude · ~/.claude/skills (직접)            ← 드물게 직접 설치
  ▢ Codex  · skills/                            ← Codex
  ◆ Gemini · (위치 미검증)                       ← 0번 검증 전이면 명시
```

- 도구 아이콘(⬣/▢/◆ 또는 디자인이 정한 글리프) + 도구 색 토큰 + 출처 텍스트.
- **플러그인에서 온 경우 "플러그인" 라벨 + 플러그인 이름을 칩(chip)으로** 분명히 표기(직접 설치와 시각 구분).
- 작은 사이즈(표 셀 안에 들어가야 함) / 패널 헤더용 약간 큰 사이즈 두 변형. 너무 길면 말줄임 + 툴팁.

## 화면 4. 개인정보(PI) 게이트 모달

**목적:** skill 본문 스캔 중 개인정보/시크릿 의심이 보이면 **멈추고** 사용자에게 확인받는다. 기본값은 보수적(제외 권장).

### 텍스트 와이어프레임

```
┌ ⚠ 민감 정보가 감지되었습니다 ───────────────────────────────┐
│ 항목: 🧩 deep-research  (출처: ⬣ Claude · 플러그인 개인)     │
│ 감지: 이메일 형태 1건 · 토큰/키 키워드 1건                   │
│ ┌ 해당 줄 (마스킹됨) ──────────────────────────────────────┐ │
│ │ 12:  contact = ****@****.com                             │ │
│ │ 37:  API_KEY = sk-****************                       │ │
│ └──────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│ 어떻게 처리할까요?                                           │
│  ○ 가명/더미로 대체   ◉ 이 항목 제외(권장)   ○ 그대로 진행  │
│  ☐ 이 출처의 비슷한 항목에도 동일 적용                       │
├─────────────────────────────────────────────────────────────┤
│ 로컬에서만 검사됩니다 · 외부 전송 없음   [취소]    [확인]    │
└─────────────────────────────────────────────────────────────┘
```

### 핵심 디자인 요구

- **모달(차단형)**: 배경 흐림 + 중앙 다이얼로그. 통과 전에는 Import가 진행되지 않음.
- **감지 요약**: 어떤 종류(이메일/토큰/키)가 몇 건인지 칩으로. **해당 줄은 반드시 마스킹**해서 표시(원문 노출 금지).
- **선택지**: 가명/더미 대체 / 이 항목 제외(기본·권장) / 그대로 진행(주의색). "그대로 진행"은 위험을 분명히 표시.
- **"로컬에서만 검사" 안내**를 모달 푸터에 상시 노출(안심 메시지).

### 상태

- **단건/다건**: 한 항목에 여러 의심이면 줄 목록을 스크롤. 여러 항목이 걸리면 "다음 항목" 페이저 또는 일괄 적용 체크박스.
- **로딩**: 마스킹/탐지 계산 중 짧은 인디케이터.
- **에러**: 마스킹 실패 시 그 줄은 "전체 가림"으로 폴백(원문 절대 노출하지 않음).

## 컴포넌트 인벤토리 (이 화면들이 쓰는 것)

- **슬라이드 패널**(우측), **모달**(차단형 다이얼로그).
- **섹션 분리 헤더**(코어 / 오버레이) + 강한 구분선.
- **키-값 행**(name/description/경로).
- **마크다운 본문 읽기 카드**(자체 스크롤, 모노스페이스 코드블록).
- **도구 오버레이 카드 ×3**(도구 색 + 상태 배지 + 메타 + 인라인 액션).
- **출처 배지 / 플러그인 칩**(표·헤더·리스트 공용, 크기 2변형).
- **상태 배지**: `✓ 최신` / `⚠ drift` / `✗ 없음` / `⚠ 손실` / `⚠ PI 의심`.
- **체크리스트 행**(체크박스 + 항목 + 출처 배지 + PI 표식).
- **충돌 카드**(라디오 선택 + diff 펼침).
- **diff 뷰**(추가/삭제 라인 색 구분, 가독성 우선).
- **액션 바**(미리보기/Import), **푸터 상태 텍스트**, **토스트**(성공/실패).

## 색 토큰 방향 (3색 = 도구별)

- **도구 3색**을 일관되게: Claude / Codex / Gemini 각각 고정 hue. 오버레이 카드·출처 배지·표 셀·라디오 옵션에서 **동일 토큰** 재사용(어느 도구인지 색만 봐도 알게).
- **상태색은 도구색과 분리된 별도 축**: 정상(중립/초록계) · 주의(앰버 — drift/손실/충돌/PI) · 위험(레드 — "그대로 진행" 같은 비가역·위험 동작) · 비활성(회색 — "없음").
- **코어 vs 오버레이 위계**: 코어는 차분한 중립 배경(원본·신뢰), 오버레이 카드는 도구 색 액센트(부가·가변)로 명도 차이. 코어가 시각적으로 "기준"으로 읽혀야 함.
- diff: 추가=초록 계열, 삭제=빨강 계열 — **대비 충분히**(가독성 최우선).

## 가독성 · 안전 강조

- **가독성 최우선**(사용자 선호): 본문/diff/마스킹 줄은 넉넉한 줄간격, 모노스페이스 코드, 충분한 대비. 정보 밀도 높은 대시보드지만 skill 본문은 "읽기 좋게".
- **안전**: PI 게이트는 차단형이고 통과 전 진행 불가. 위험 옵션("그대로 진행")은 색·문구로 분명히. **마스킹된 줄 외 원문 노출 금지.**
- **정직한 손실/부재 표시**: 없는 것은 "없음", 변환 불가/미검증은 `⚠`로 — **임의로 채우지 않는다**(예: Gemini skill 위치 미검증이면 "위치 미검증" 그대로 표기).
- **2단계 쓰기 존중**: 이 화면들의 액션은 미리보기/가져오기 위주이며, 실제 push 적용은 `screen-design.md`의 2단계 확인 흐름으로 넘긴다.

## 반응형

- **넓은 화면**: 통합 표 + 우측 슬라이드 상세 패널 동시. 오버레이 카드 3개 가로 정렬.
- **좁은 화면**: 상세 패널이 전체 폭 오버레이로 전환. 오버레이 카드 3개는 세로 스택. Import 충돌 카드의 라디오/ diff도 세로로 흐름.
- **모달(PI 게이트)**: 모든 폭에서 중앙 정렬, 좁은 화면에서는 거의 전체 폭. 본문 마스킹 줄·diff는 가로 스크롤 컨테이너로 감싸 페이지가 가로로 밀리지 않게.
- 출처 배지·플러그인 칩은 좁아지면 아이콘+말줄임 + 툴팁으로 축약.
