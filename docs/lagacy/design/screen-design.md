# 화면 설계 (Screen Design)

이 문서는 나중에 **Claude Design**에 넘길 입력이다. 프로토타입은 스타일 없는 기능용
HTML이고, 여기 적힌 화면 구성·상태·상호작용을 기준으로 Claude Design이 외형을 입힌다.

핵심 UX 원칙(설계 결정에서 옴):
- **한눈에 보기:** 세 도구 × 세 타입의 존재/부재/차이(drift)를 한 화면에.
- **안전 우선:** 모든 쓰기는 `미리보기(diff) → 적용` 2단계. 적용 전 자동 백업.
- **정직한 손실 표시:** 변환 불가 항목은 `⚠ 미지원`으로 표시(임의로 안 채움).
- **개인정보 게이트:** 민감 의심 항목은 멈추고 사용자에게 확인.

---

## 화면 1. 메인 통합 뷰 (홈)

**목적:** 모든 항목이 어느 도구에 있고/없고, canonical과 다른지(drift)를 한눈에.

```
┌─────────────────────────────────────────────────────────────┐
│  AI-Agent Dashboard            [새로고침]   서버: ● 127.0.0.1 │
│  필터: (전체) [skill] [agent] [지시문]      검색: [________]   │
├──────────────────┬────────┬────────┬────────┬───────────────┤
│ 항목 (canonical) │ Claude │ Codex  │ Gemini │ 상태          │
├──────────────────┼────────┼────────┼────────┼───────────────┤
│ 📄 global 지시문 │  ✓     │  ✓     │  ✓     │ ✅ 동기화됨    │
│ 📄 project 지시문│  ✓     │  ⚠drift│  ✗없음 │ 🟠 차이 있음   │
│ 🧩 code-review   │  ✓     │  ✓     │  ✗없음 │ 🟠 일부 누락   │
│ 🤖 reviewer agent│  ✓     │ ✓.toml │  ⚠손실 │ 🟠 변환 손실   │
└──────────────────┴────────┴────────┴────────┴───────────────┘
                                          [모두 미리보기]  …
```

- **행:** canonical 항목 1개. 타입 아이콘(📄지시문/🧩skill/🤖agent).
- **열:** Claude·Codex·Gemini별 상태 셀 — `✓ 동일` / `⚠ drift(다름)` / `✗ 없음` / `⚠ 손실(변환불가 필드)`.
- **상태 배지:** ✅동기화됨 / 🟠차이있음 / 🔴오류.
- **상단:** 새로고침(=`status` 재호출), 타입 필터, 검색, 서버 연결 표시.
- **상태들:** 로딩(스켈레톤), 빈 목록(“아직 가져온 항목 없음 → Import 안내”), 서버 꺼짐(정적 보기 배너).
- **하이브리드 읽기(D9):** 기본 화면은 *존재/부재*를 자동 표시. **drift·내용은 항목을 열 때** 계산·표시(자동으로 전체 내용을 읽지 않음).

## 화면 2. 항목 상세 패널 (행 클릭 시 우측 슬라이드)

**목적:** 한 항목의 canonical 내용과 도구별 사본 상태를 자세히.

```
┌ reviewer agent ──────────────────────────────[×]┐
│ id: agent-reviewer   타입: agent   scope: global │
│ 출처: codex          수정: 2026-06-30            │
├──────────────────────────────────────────────────┤
│ canonical (중립 스키마)        | 도구별 상태       │
│  name: reviewer                | Claude  ✓ 최신    │
│  model: ...                    | Codex   ✓ 최신    │
│  tools: [Read, Grep, Bash]     | Gemini  ⚠ 손실    │
│  system_prompt: "..."          |   └ Bash 매핑 없음 │
├──────────────────────────────────────────────────┤
│ [Gemini로 미리보기] [Codex로 미리보기] [Import]   │
└──────────────────────────────────────────────────┘
```

- 좌: canonical 원본. 우: 각 도구 사본의 최신/drift/손실 + **무엇이 손실됐는지** 구체 표시.
- 액션: 특정 도구로 미리보기(→화면 3), Import(→화면 4).

## 화면 3. Sync(push) 흐름 — 미리보기 → 적용

**목적:** 변환 결과를 먼저 보고, 명시적으로 한 번 더 눌러야 적용.

```
1단계 미리보기            2단계 적용 확인
┌ Codex로 동기화 ───────┐  ┌ 적용하시겠어요? ──────┐
│ diff:                 │  │ 대상: ~/.codex/...    │
│  - 이전 줄 (빨강)      │  │ 백업: backups/2026..  │
│  + 새 줄 (초록)        │  │ ⚠ 변환 손실 1건       │
│ ⚠ 미지원: temperature │  │  [취소]   [적용 실행] │
│  [취소] [적용하기]→    │  └──────────────────────┘
└───────────────────────┘
적용 후: ✅ 완료 + 백업 경로 + “status 새로고침” 토스트
```

- **diff 패널:** 추가/삭제 라인 색 구분. 변환 손실은 패널 상단에 ⚠로 요약.
- **2단계 확인:** 대상 경로 + 백업 경로 + 손실 건수 표시 후에야 “적용 실행”.
- **상태:** 적용 성공/실패(실패 시 원인 + 백업으로 복구 안내).

## 화면 4. Import(pull) 흐름 + 충돌 해결

**목적:** 각 도구에 흩어진 기존 항목을 canonical로 가져오기(역방향).

```
┌ Import: 가져올 항목 선택 ─────────────────────────┐
│ [✓] Codex/skills/code-review                      │
│ [✓] Codex/agents/reviewer.toml                    │
│ [ ] Claude/CLAUDE.md (글로벌)                      │
├───────────────────────────────────────────────────┤
│ ⚠ 충돌: 'code-review'가 Claude·Codex 양쪽에 있음  │
│   ◉ Claude 버전 사용  ○ Codex 버전 사용  ○ 둘 다 보존│
├───────────────────────────────────────────────────┤
│                         [취소]   [canonical로 가져오기]│
└───────────────────────────────────────────────────┘
```

- 같은 이름이 여러 도구에 있으면 **충돌 카드**로 사용자가 선택.
- 가져오기 전 PI 게이트(화면 5) 통과.

## 화면 5. 개인정보 확인 게이트 (모달)

**목적:** 스캔 본문에서 개인정보/시크릿 의심이 보이면 멈추고 확인(D7).

```
┌ ⚠ 민감 정보가 감지되었습니다 ─────────────────────┐
│ 항목: Claude/CLAUDE.md                            │
│ 감지: 이메일 형태 1건, 토큰 키워드 1건            │
│ 해당 줄(마스킹):  ...  ****@****.com  ...         │
├───────────────────────────────────────────────────┤
│ 어떻게 처리할까요?                                 │
│  ○ 가명/더미로 대체   ○ 이 항목 제외   ○ 그대로 진행│
│                         [취소]        [확인]       │
└───────────────────────────────────────────────────┘
```

- 감지 종류·해당 줄(마스킹)·선택지(대체/제외/그대로) 제공. 기본값은 보수적(제외 권장).

## 화면 6. (Phase 2 예약 영역 — 지금은 자리만)

나중에 붙일 화면들. 프로토타입에선 비활성 탭/플레이스홀더로만 둔다.

- **사용량·상태 뷰:** 항목별 사용량 배지(편집 churn / 호출 횟수), `active/stale/archived` 상태, **신호 출처(A=git·mtime / B=로그)** 구분 표시.
- **최근 미사용(LRU) 목록:** 오래 안 쓴 항목 순.
- **제안 인박스:** “X·Y 겹침 → 통합?”, “자주 수정됨 → 새 버전?” 같은 제안 카드.
- **대기 변경(staging):** 제안의 diff + [승인]/[거절]. 자동 적용 없음.
- **리포트 뷰:** `review` 실행 결과(`run.json`/`REPORT.md`) 타임라인.

## 디자인 힌트 (Claude Design용)

- 정보 밀도 높은 **대시보드 표** 중심. 상태는 색+아이콘으로 빠르게 스캔.
- 도구 3개는 일관된 컬러 토큰(Claude/Codex/Gemini)으로 구분.
- 위험 동작(적용/Import)은 명확한 2단계와 색으로 구분, diff는 가독성 최우선(사용자 선호=가독성).
- 로컬 전용 도구임을 헤더에 표시(외부 전송 없음).
