# 0번 검증 — 에이전트 개념 재검증 (UX v2 F0)

- **검증일**: 2026-07-02
- **목적**: UX 대개편 v2에서 에이전트 뷰를 설계하기 전에, "Gemini는 에이전트 개념이 없는가?", "에이전트는 여러 참고 파일을 갖는 폴더 구조인가?", "Claude↔Codex 에이전트는 호환 가능한가?"를 **공식 문서 + 로컬 실물** 양쪽으로 재확인.
- **방법**: 각 도구의 공식 문서(웹)를 직접 조회해 로컬 실물 구조와 대조. (보고에는 실제 파일 본문·개인정보를 싣지 않음 — 경로·구조·개수만.)

## 결론 요약

| 질문 | 결론 | UI 반영 |
|---|---|---|
| Gemini에 에이전트 개념이 없나? | **아니오 — 공식 지원됨.** Gemini CLI 서브에이전트는 `.gemini/agents/*.md`(프로젝트)·`~/.gemini/agents/*.md`(홈)에서 로드되며 현재 기본 활성(`experimental.enableAgents` 토글로 존재). | **Gemini 열 유지.** 로컬에 폴더가 없는 것은 "미사용"일 뿐 "미지원"이 아님 → 셀에 "미사용" 은은한 표시. |
| 에이전트 = 여러 참고 파일 폴더인가? | **아니오 — 세 모델 모두 공식 근거 없음.** 세 도구 공통으로 에이전트 정의는 **단일 파일**(frontmatter + 시스템 프롬프트). "폴더에 참고 파일 여러 개"는 어느 공식 문서에도 없음. | **폴더형 UI 없음** — 단일 파일 비교 유지. 대신 공식 근거가 있는 **에이전트→스킬 참조**를 노출. |
| 에이전트가 다른 파일을 참조하는 공식 방법은? | **스킬 참조.** Claude Code: 에이전트 frontmatter `skills:` 목록. Codex: `skills.config` 설정. 이것이 "에이전트가 여러 리소스를 쓰는" 공식 메커니즘. | 에이전트 상세에 **"참조 스킬" 섹션** — 파싱해 링크로 표시, 클릭 시 스킬 뷰 해당 항목으로 이동. |
| Claude↔Codex 에이전트 호환? | **변환 가능(현행 transform/agent.js 가정 유지).** 양쪽 다 단일 md/설정 기반이며 중립 스키마(name/description/system_prompt/model/tools)로 상호 변환된다. 단 도구 이름 매핑 손실은 기존과 동일하게 표시. | 기존 변환 파이프라인 유지. |
| 로컬 `~/.codex/agents/*.md` 실물은? | **비표준.** Codex가 읽는 공식 위치가 아님(Codex는 이 경로의 .md를 로드하지 않음). 과거 수동 배치로 추정. | Codex 열에서 해당 실물은 **"비표준(모델이 읽지 않음)"** 표시. |

## 세부 확인 내용

### 1) Gemini CLI 서브에이전트
- 공식 문서 기준 서브에이전트 정의 파일은 마크다운(frontmatter 포함)이며 로드 경로는 프로젝트 `.gemini/agents/` 와 홈 `~/.gemini/agents/`.
- 초기에는 실험 플래그(`experimental.enableAgents`) 뒤에 있었고 현재는 기본 활성 상태로 확인됨(플래그는 끄기용으로 잔존).
- 로컬 실기기에는 `~/.gemini/agents/` 폴더 자체가 없음 = **미사용 상태**. 스캐너가 "폴더 없음"을 "미지원"으로 오해하지 않도록 UI 문구를 "미사용"으로 구분한다.

### 2) 폴더형(다중 참고 파일) 에이전트
- Claude Code: `.claude/agents/<name>.md` 단일 파일(frontmatter: name/description/tools/model 등). 폴더 구조 근거 없음.
- Codex: 에이전트류 설정은 config 기반(단일 정의). 폴더형 근거 없음.
- Gemini: 위와 같이 단일 md. 폴더형 근거 없음.
- → **UX v2에서 "에이전트 폴더 뷰"는 만들지 않는다.** 사용자가 원했던 "여러 참고 파일"의 실체는 **스킬 참조**로 공식 지원되므로 그것을 노출한다.

### 3) 에이전트 → 스킬 참조 (공식 근거 있음)
- Claude Code: 에이전트 frontmatter의 `skills:` 필드로 스킬 목록을 지정할 수 있음.
- Codex: `skills.config`를 통해 스킬 구성이 가능.
- → 에이전트 상세 패널에 "참조 스킬" 섹션을 추가하고, 각 항목을 스킬 뷰로 링크(F5에서 core 파싱 헬퍼 소폭 추가).

### 4) 로컬 실물 대조 (요약만)
- `~/.claude/agents/`: 표준 위치, 단일 .md 구조 일치.
- `~/.codex/agents/*.md`: **공식 로드 경로 아님** — Codex 문서 어디에도 이 경로를 읽는다는 근거 없음. 대시보드에는 존재를 표시하되 "비표준" 배지.
- `~/.gemini/agents/`: 부재 = 미사용(지원은 됨).

## UX v2 반영 사항 (F5 설계 확정)

1. 에이전트 매트릭스에 **Gemini 열 유지** — 값 없으면 "미사용".
2. Codex 열의 비표준 실물 → "비표준(모델이 읽지 않음)" 표시.
3. 폴더형 UI 없음 — 단일 파일 비교 유지.
4. 상세 패널에 "참조 스킬" 섹션(Claude `skills:` / Codex `skills.config` 파싱).
